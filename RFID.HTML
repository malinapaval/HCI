Definition

A radio frequency identification system or RFID uses radio waves to read and capture various information stored on a tag attached to an object.

An RFID is made up of a reader, a set of tags and a communication protocol between the reader and the tags. A tag is a transmitter and a responder whose resources are considered to be limited.

R.F.I.D-1

From a formal point of view, an RFID system can be defined as the following:
• R an identifier for the reader
• T has a set of labels for the reader, whose size is polynomial

An RFID system over (R, T) is a triplet S = (SetupR, SetupT, Ident)
• SetupR (range) - takes a range as a parameter and returns a triplet (pk, sk, DB) consisting of a key pair (pk, sk) and an empty D database. It should be noted that pk is a public key while sk is a key kept secret by the reader

R.F.I.D-2

SetupT (pk, ID) - initializes the tag identified with ID and returns an S state of the initial tag and a secret K specific to the same tags. The pair consisting of ID and secret K is stored in the DB

Ident (pk; R (sk, DB); ID (s)) - represents the communication protocol between a reader, R, with its own secret key sk and the database DB, and a tag identified by an ID, with the status S. Following this protocol, the reader can receive either the tag ID or the | symbol.

Security

Black box model - this model allows the opponent to observe the response returned by the system when he is the one who gives it inputs, it is even possible that the opponent knows the algorithms that are used in the cryptographic system.

Gray box model - in addition to the properties offered by the black box model, the gray box allows the opponent to have access to some information such as energy consumption or electromagnetic radiation emitted.

Oracles-1

1. CreateTagb (ID) - the SetupT algorithm (pk, ID) is called to generate a pair (K, S) and to create a T tag with the specified ID and the initial state S. If the bit is 1 then the created label is legitimate. The label is considered free

2. DrawTag (si) - through this oracle the opponent is allowed to interact with free tags according to a certain probability distribution
It is noted that this oracle provides information on whether or not the tag is legitimate.

Oracles-2

3. Free (vtag) - using this oracle the opponent can release the tag tied to the vtag. The vtag identifier will no longer be usable, as its temporary state is deleted when it is released.

4. Launch () - calling this oracle means that the opponent wants to launch a new protocol, in other words this oracle returns a unique identifier that will be used with the protocol.

Oracles-3

5. SendReader (m, pi) - through this oracle, the opponent receives the reader's response when a message m is part of a pi protocol. When the message m is empty, the oracle returns the first message from the pi protocol.

6. SendTag (m, vtag) - this oracle returns the tag response when the m message is sent to the tag represented by the vtag. When the m message is blank this oracle returns the first message from the pi protocol

Oracles-4

7. Result (pi) - this oracle allows the opponent to know the reader's decision regarding the authentication of the tag in the pi session. The oracle will return 1 if during the session the reader authenticated the tag and 0 in the opposite case.

8. Corrupt (vtag) - this oracle returns the permanent state of the label represented by the vtag, when the label is not part of any calculation of any step

Adversaries

Weak opponents - no access to Corrupt Oracle ()
Direct opponents - if they have access to the Corrupt Oracle then that is the only one they can access

Destructive opponents - after obtaining information from the Corrupt Oracle (vtag), the opponent destroys the tag identified by the vtag and it will no longer be available, the database will still maintain the existence of that tag
