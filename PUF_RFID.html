Definition

A PUF-based RFID scheme is an RFID scheme that contains PUF type tags.

There are two approaches to the authentication protocol for RFID schemes based on PUFs.

First Case-1

The first approach perceives PUFs as fingerprints, initially requiring the configuration of a phase in which a PUF model or a varied set of pairs (challenge-response) are pre-calculated and stored in the reader's database.

To identify a tag, the reader queries the database by a challenge, the tag evaluates its function on that challenge and then compares it with the value already stored in the database.

First Case-2

This model does not guarantee the type of security we want, as there is a possibility that an opponent may have enough challenge-response pairs to be able to simulate the PUF of the label.

Also, a strong opponent could run the authentication protocol with a tag until it exhausts all the challenge-response pairs stored in the database, in which case either a pair will be reused or a reset mechanism must be introduced.

Second Case

The second approach is based on the idea of ​​using PUFs as cryptographic key generators or as storage methods.

A tag will evaluate its PUF only to generate or extract a cryptographic key. Thus, the function is evaluated for a minimum number of attempts, thus eliminating the possibility that an opponent may model a PUF.

Uses

In order to use a RFID scheme based on a PUF in Vaudney's model, we need to clarify what corruption means in this case, in order to be able to make the necessary adaptations.

There are two types of scenarios.

First Use

By corrupting a PUF tag, the opponent obtains the status of the tag, in connection with the attack model. In addition, the tag is destroyed, but its function can still be evaluated.

This variant does not offer major differences compared to the case of corruption of normal tags (which are not incorporated in an RFID scheme) because the function of the tag can be seen as a public function that the opponent can evaluate as he wishes.

Second Use

By corrupting the tag, the opponent obtains the status of the tag in relation to the type of attack model.

Both the tag and its PUF are destroyed, in which case the PUF can no longer be evaluated.
